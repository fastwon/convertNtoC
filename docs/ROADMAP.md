# convertN2C — 단계별 개발 로드맵

> **진행 상황:** P0 ✅ · P1 ✅ · P2 ✅ · P3 ✅ · P4 진행중(P4a 회차업로드 ✅ · P4b LLM추상화+Gemini+무료토글 ✅ · 다음 P4c 인물추출)

> 기반: [CLAUDE.md](../CLAUDE.md) · [docs/DESIGN.md](./DESIGN.md)
> 대상: Windows 데스크톱 EXE (PyWebView + PyInstaller, Python/FastAPI 로컬 + React SPA, LLM=Claude, 이미지=외부 API, 타인 배포·사용자 자기 키).
>
> 원칙: **각 단계는 독립적으로 "완료 기준"을 만족하면 끝.** 위에서 아래로 의존하므로 순서대로 진행. P0~P2는 기반 공사, P3부터 기능. P4 끝나면 "텍스트만이라도 도는" 첫 세로 슬라이스가 나오고, P6 끝나면 실제 만화 컷이 나온다.

## P0 — 프로젝트 스캐폴딩 & 빈 껍데기 exe
**목표:** 아무 기능 없어도 `convertN2C.exe`가 빌드되고, 창이 뜨고, 그 안에서 React가 FastAPI(127.0.0.1)와 통신하는 골격 완성.

- [ ] 레포 구조 확정: `app/`(Python: api/llm/services/storage), `frontend/`(React SPA), `packaging/`(PyInstaller spec)
- [ ] Python 환경(venv/uv)·의존성 관리, 린트/포맷터(ruff 등) 설정
- [ ] FastAPI를 127.0.0.1에 in-process 기동 + `/health` 엔드포인트
- [ ] React(Vite) SPA 초기화, 정적 빌드 산출
- [ ] PyWebView 창이 빌드된 SPA를 로드, SPA → `/health` 호출 성공
- [ ] PyInstaller one-file 빌드: 정적 자산 포함, `sys._MEIPASS` 기준 경로 해석
- [ ] WebView2 런타임 부재 시 안내 처리(확인만)

**완료 기준:** 더블클릭한 exe에서 창이 뜨고 "백엔드 연결됨"이 화면에 보인다.

## P1 — 설정 & API 키 관리
**목표:** 사용자가 본인 키를 안전하게 넣고 검증하는 흐름. (FR-0.0)
*의존: P0*

- [ ] 설정 화면 UI: Anthropic 키 / 이미지 API 키 입력
- [ ] `keyring`(Windows Credential Manager) 저장·로드. 평문 파일·번들 금지
- [ ] 키 검증: Anthropic은 가벼운 호출(`models.list` 등), 이미지 API는 ping
- [ ] 키 없음/만료/무효 시 설정 화면으로 유도하는 가드
- [ ] 로그·예외 메시지에 키 노출 차단(마스킹) 확인

**완료 기준:** 키를 넣으면 "유효함"이 뜨고 재시작 후에도 유지된다. 잘못된 키는 명확히 거부된다.

## P2 — 로컬 데이터 계층
**목표:** 모든 영속 데이터의 로컬 저장 기반. (S3·클라우드 없음)
*의존: P0*

- [ ] 앱 데이터 루트 `%APPDATA%\convertN2C\` (`platformdirs`)
- [ ] SQLite 스키마 + 마이그레이션: Project / Character / Episode / Panel / GlobalMemory
- [ ] 데이터 접근 계층(repository) — 모든 쿼리 `project_id` 스코프
- [ ] 로컬 벡터 저장소 연동(sqlite-vec 권장): 임베딩 저장/검색 인터페이스
- [ ] 파일 저장소 헬퍼: `projects\<project_id>\` 하위 이미지 입출력
- [ ] 백업/초기화 고려(최소: DB 파일 위치 노출)

**완료 기준:** 더미 데이터로 CRUD + 벡터 upsert/검색이 로컬 파일에 영속된다.

## P3 — 프로젝트 & 대시보드 & 스타일 고정
**목표:** 다중 프로젝트 관리와 글로벌 스타일 에셋 고정. (FR-0.1, FR-0.2)
*의존: P2*

- [ ] 대시보드: 프로젝트 목록/생성/삭제
- [ ] 프로젝트 생성 시 화풍 스타일 지정(스타일 프롬프트 + 이미지 모델/LoRA 식별자 필드)
- [ ] 스타일을 프로젝트의 기본값으로 고정 — 이후 모든 회차 생성의 강제 기본값
- [ ] 프로젝트별 폰트/말풍선 설정 자리(값만 저장, 합성은 P7)

**완료 기준:** 여러 프로젝트를 만들고 각각 다른 화풍이 저장·재진입된다.

## P4 — LLM 클라이언트 + 회차 업로드 + 인물 추출/교차검증
**목표:** 첫 세로 슬라이스(텍스트). 소설을 넣으면 인물이 추출되고 캐릭터 뱅크가 쌓인다. (FR-1.1, FR-0.3, FR-1.2)
*의존: P1, P2, P3*

- [ ] `app/llm/` 의 `LLMProvider` 추상화 + **Claude 구현체**(`claude-api` 스킬 기준) + **Gemini 무료 구현체**
- [ ] 설정에 "무료 버전 사용" 토글 + Gemini 키 슬롯(P1 키 관리 확장)
- [ ] 모델-역할 매핑(Claude): 분류=haiku, 요약/추출=sonnet, 추론=opus
- [ ] 회차(Episode) 업로드: 텍스트 입력 + (선택) Files API 업로드
- [ ] 인물 추출 + 신규/기존 교차검증: `output_config.format`(JSON schema)로 `{characters:[{name,is_new,traits,matched_character_id}]}`
- [ ] 유저 확인 UI: 신규/기존 판별 결과 확인·수정
- [ ] 캐릭터 뱅크 저장: 참조 이미지 등록(로컬 파일) + 특징/임베딩(벡터 저장소)

**완료 기준:** 1화 업로드 → 인물 목록 추출 → 참조 이미지 등록 → 2화 업로드 시 기존 인물로 인식된다.

## P5 — 글로벌 메모리 & 컨텍스트 & 프롬프트 캐싱
**목표:** 회차 간 일관성의 핵심. 글로벌 메모리 조립과 캐싱, 회차 요약 누적. (FR-0.4)
*의존: P4*

- [ ] 글로벌 메모리 조립기: 스타일 + 캐릭터 뱅크 + 세계관 + 누적 회차 요약
- [ ] 프롬프트 캐싱: 글로벌 메모리를 system 프리픽스 + `cache_control`(고정 앞/변동 뒤). 캐시 무효화 패턴(now/UUID) 회피
- [ ] 회차 요약 생성(sonnet) → 누적 저장(원문 아닌 요약으로 토큰 관리)
- [ ] 다음 회차 콘티 생성 시 맥락 주입(이전 사건/관계 변화)
- [ ] (검증) `usage.cache_read_input_tokens`로 캐시 적중 확인

**완료 기준:** 2화 생성 입력에 1화 요약·캐릭터·스타일이 자동 주입되고, 반복 호출에서 캐시 적중이 관측된다.

## P6 — 콘티/프롬프트 엔진 + 이미지 생성
**목표:** 실제 만화 컷 산출. (FR-2.1)
*의존: P5 · 이미지 API 공급자 결정 필요*

- [ ] LLM 콘티 생성: 회차 → 패널(컷) 분할 + 각 컷 묘사
- [ ] 결정적 프롬프트 엔진: `[스타일]+[캐릭터 뱅크]+[컷 묘사]` 최종 문자열을 코드로 조립
- [ ] `ImageGenerator` 인터페이스 + 외부 API 1차 구현체(키는 keyring)
- [ ] 캐릭터 일관성 적용(외부 API의 IP-Adapter/InstantID·LoRA 기능 연동)
- [ ] 생성 컷을 로컬 파일 저장소에 저장 + 패널 메타 기록
- [ ] 진행률/에러 UX(streaming, 실패 재시도)

**완료 기준:** 한 회차에 대해 고정 화풍·일관 캐릭터의 패널 이미지들이 생성·저장되어 화면에 보인다.

## P7 — 대사 & 레이아웃 합성
**목표:** 컷에 대사/말풍선을 얹어 "만화" 완성도. (FR-2.2)
*의존: P6*

- [ ] 컷별 대사 배치(JSON) + 말풍선 렌더링
- [ ] 프로젝트별 지정 폰트/말풍선 스타일 적용
- [ ] 페이지/컷 레이아웃 조합·미리보기
- [ ] 내보내기(이미지/페이지 묶음)

**완료 기준:** 대사 입힌 완성 컷을 회차 단위로 내보낼 수 있다.

## P8 — 배포 마감 & 다듬기
**목표:** 타인 배포 가능한 수준의 안정화.
*의존: 전체*

- [ ] PyInstaller 최종 빌드(아이콘/버전/용량 점검), WebView2 부재 안내·설치 가이드
- [ ] 비용 투명성: 호출 전/후 예상·실사용 토큰·이미지 수 표시
- [ ] 스토리지 관리 UI(용량 표시·프로젝트 삭제·DB 위치)
- [ ] 오프라인/키 만료/요금 한도 등 에러 메시지 정리
- [ ] (선택) 자동 업데이트 방식 결정·적용
- [ ] 간단 사용 설명서(첫 실행·키 발급 가이드)

**완료 기준:** 키만 발급받은 새 사용자가 설치→키 입력→1·2화로 일관된 만화 생성까지 막힘없이 진행한다.
